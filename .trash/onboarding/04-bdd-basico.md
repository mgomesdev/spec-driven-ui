# BDD Básico: Entendendo Behavior-Driven Development

## O Que É BDD?

BDD é uma forma de escrever requisitos que **todos entendem** — PO, Dev, QA, até o cliente.

```
BDD = Given-When-Then em português estruturado

Given  = Dado que    (Contexto)
When   = Quando      (Ação)
Then   = Então       (Resultado esperado)
```

---

## Anatomia de Um Cenário

### Exemplo Prático: Header Navigation

```gherkin
# specs/features/header/header.feature

Feature: Header Navigation

  @desktop @pending
  Scenario: Logo click redirects to home
    Given I am on any page
    When I click the logo
    Then I should be on the home page
```

###分解 (Decomposição)

```
┌─────────────────────────────────────────────────────────────────┐
│                    ANATOMIA DO CENÁRIO                          │
└─────────────────────────────────────────────────────────────────┘

Feature: Header Navigation          ← TEMA (agrupa cenários)
│
├── @desktop                        ← TAG (contexto)
│
└── Scenario: Logo click...         ← TÍTULO (o que testar)
    │
    ├── Given I am on any page      ← CONTEXTO (onde estamos)
    │
    ├── When I click the logo       ← AÇÃO (o que fazer)
    │
    └── Then I should be on home    ← RESULTADO (o que esperar)
```

## Tags: Categorizando Cenários

### Tags Principais

| Tag | Uso | Exemplo |
|-----|-----|---------|
| `@desktop` | Testes desktop | Header em 1280px+ |
| `@mobile` | Testes mobile | Menu hamburger |
| `@tablet` | Testes tablet | Menu responsivo |
| `@a11y` | Acessibilidade | Navegação por teclado |
| `@pending` | Não implementado | Sua tarefa! |
| `@done` | Implementado | Pronto para PR |

### Usando Múltiplas Tags

```gherkin
@desktop @a11y @pending
Scenario: User can navigate using keyboard
  Given I am on the home page
  When I press Tab three times
  Then the logo should be focused
```

## Diagrama: Como Um Cenário Vira Teste

```
┌─────────────────────────────────────────────────────────────────┐
│              TRANSFORMAÇÃO: CENÁRIO → TESTE                      │
└─────────────────────────────────────────────────────────────────┘

   *.feature                              frontend/tests/
   ─────────                              ───────────────
   
   Given I am on any page          →    await page.goto('/')
   
   When I click the logo           →    await page.click('[data-testid="logo"]')
   
   Then I should be on home        →    await expect(page).toHaveURL('/')
```

## Exemplo Completo: Header Feature

### O *.feature

```gherkin
# specs/features/header/header.feature

Feature: Header Component

  Background:
    Given the viewport is desktop size

  @desktop @pending
  Scenario: Logo click navigates to home
    Given I am on the "/about" page
    When I click the header logo
    Then I should be navigated to "/"

  @desktop @pending  
  Scenario: Desktop menu shows all navigation items
    Given I am on the home page
    Then I should see the following navigation items:
      | Home     |
      | About    |
      | Projects |
      | Contact  |

  @mobile @pending
  Scenario: Mobile menu opens on hamburger click
    Given I am on the home page
    And the viewport is mobile size
    When I click the hamburger menu button
    Then the mobile overlay should be visible
```

### O Teste Playwright (autogenerated)

```typescript
// frontend/tests/features/header/US-001.spec.ts

import { test, expect } from '@playwright/test';

test.describe('Header Component', () => {

  test.beforeEach(async ({ page }) => {
    await page.setViewportSize({ width: 1280, height: 720 });
  });

  test('Logo click navigates to home', async ({ page }) => {
    // Given I am on the "/about" page
    await page.goto('/about');
    
    // When I click the header logo
    await page.click('[data-testid="header-logo"]');
    
    // Then I should be navigated to "/"
    await expect(page).toHaveURL('/');
  });

  test('Desktop menu shows all navigation items', async ({ page }) => {
    // Given I am on the home page
    await page.goto('/');
    
    // Then I should see the following navigation items
    await expect(page.locator('[data-testid="header-nav-0"]')).toHaveText('Home');
    await expect(page.locator('[data-testid="header-nav-1"]')).toHaveText('About');
    await expect(page.locator('[data-testid="header-nav-2"]')).toHaveText('Projects');
    await expect(page.locator('[data-testid="header-nav-3"]')).toHaveText('Contact');
  });

  test('Mobile menu opens on hamburger click', async ({ page }) => {
    // Given the viewport is mobile size
    await page.setViewportSize({ width: 375, height: 667 });
    await page.goto('/');
    
    // When I click the hamburger menu button
    await page.click('[data-testid="hamburger-button"]');
    
    // Then the mobile overlay should be visible
    await expect(page.locator('[data-testid="header-mobile-overlay"]')).toBeVisible();
  });
});
```


## Boas Práticas de BDD

### ✅ Faça

- Use linguagem de negócio, não técnica
- Um cenário = uma unidade de comportamento
- Given-When-Then em uma linha lógica
- Tags para categorizar e filtrar

### ❌ Não Faça

- Não coloque implementação no cenário
- Não use "e" excessivamente (máx 2 por passo)
- Não teste múltiplos comportamentos juntos
- Não use linguagem técnica (ex: "clica no <a>")

## Exercício Prático

### Sua Tarefa

Transforme este requisito em cenário BDD:

**Requisito:** "Quando o usuário clica em 'Login' no header, deve aparecer um modal de login com campos de email e senha."

### Solução

```gherkin
@desktop @pending
Scenario: Login modal opens on header button click
  Given I am on the home page
  When I click the "Login" button in the header
  Then the login modal should be visible
  And the modal should contain an email input field
  And the modal should contain a password input field
```
